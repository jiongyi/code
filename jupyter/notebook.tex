    \section{Getting started with Jupyter
notebook}\label{getting-started-with-jupyter-notebook}

Jupyter can be installed using
\href{https://www.anaconda.com/}{Anaconda}, a Python data science
platform that includes the most popular science libraries and makes
their management pretty straightforward.

\subsection{Installation}\label{installation}

If you have a Mac, then follow this
\href{https://docs.anaconda.com/anaconda/install/mac-os}{link} to
install Anaconda. Jupyter will be installed by default. If you need to
install, remove, or manage other packages, refer to the conda user
\href{https://conda.io/docs/user-guide/getting-started.html}{guide}.

\subsection{Accessing Jupyter
notebooks}\label{accessing-jupyter-notebooks}

\paragraph{Locally}\label{locally}

A Jupyter notebook server can be started by issuing the following
command in a terminal:

\begin{verbatim}
jupyter notebook
\end{verbatim}

By default, the server can be accessed at http://localhost:8888 via your
web browser of choice.

\paragraph{Remotely}\label{remotely}

I have setup a Jupyter notebook server so that it can be accessed at
https://jupyter.ucsf.edu:8888. The domain name can be resolved only by
UCSF's DNS servers, so you must connect to UCSF's VPN. The connection
has been secured using an SSL certificate, and the server password is
"serpiente y culebra" (without quotes). Jupyter was not designed to have
a multi-user environment, so overwriting other people's work when they
are logged on is a concern.
\href{https://jupyterhub.readthedocs.io/en/latest/}{JupyterHub} is a
possible solution, but I'm still working on setting the server up
properly.

\subsection{Using Jupyter notebooks}\label{using-jupyter-notebooks}

Jupyter notebooks feel like Mathematica notebooks but are much more
versatile. Notebook contents are organized into cells; in one type you
can use markdown to format text on the fly (LaTeX is also supported),
and in another type you can run code using a variety of kernels
supported by Jupyter. The server at jupyter.ucsf.edu supports only
Python at the moment. This
\href{https://jupyter-notebook.readthedocs.io/en/stable/ui_components.html}{link}
explains how the user interface is setup
